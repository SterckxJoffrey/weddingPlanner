Projet MERN : Wedding Planner

L’objectif de cet exercice est de vérifier que votre backend est bien connecté à votre frontend, que les principales fonctionnalités du projet Wedding Planner vu en classe fonctionnent correctement, et que vous êtes capables de documenter votre travail de manière professionnelle.

Travail demandé : 

À partir du projet Wedding Planner expliqué et démontré en classe, vous devez vérifier, compléter si nécessaire, puis démontrer le bon fonctionnement des fonctionnalités suivantes :

1. Authentification :

Vous devez montrer que l’application possède :

une page d’inscription fonctionnelle
une page de connexion fonctionnelle

L’utilisateur doit pouvoir :

créer un compte
se connecter avec ses identifiants
accéder aux pages protégées une fois connecté
2. Vérification de la liaison Backend / Frontend

Vous devez prouver que le frontend React communique correctement avec le backend Express / Node.js.

Pour cela, vous devez montrer :

avec quel moyen la communication est faite 
quelles routes API sont utilisées
si les données envoyées depuis le frontend sont bien enregistrées dans la base de données
3. Gestion des mariages :

Vous devez permettre à un utilisateur connecté de :

ajouter un mariage
afficher la liste des mariages existants
4. Gestion des invités :

Vous devez permettre à un utilisateur connecté de :

ajouter un invité
lier cet invité à un mariage existant
5. Gestion des prestataires :

Vous devez permettre à un utilisateur connecté de :

ajouter un prestataire
lier ce prestataire à un mariage existant
Ce que vous devez expliquer :

Dans votre documentation, vous devez expliquer clairement :

comment fonctionne votre architecture
comment le frontend parle avec le backend
comment les données sont stockées dans MongoDB
comment vous avez vérifié que tout fonctionne
quels problèmes vous avez rencontrés et comment vous les avez corrigés
Livrables attendus
1. Dépôt GitHub :

Vous devez remettre un lien GitHub contenant :

le frontend
le backend
un fichier README.md clair, structuré et bien expliqué
2. README.md :

Votre README doit contenir au minimum :

le nom du projet
l’objectif du projet
les technologies utilisées
les étapes d’installation
la configuration des fichiers .env
comment lancer le backend
comment lancer le frontend
les routes principales de l’API
un résumé du fonctionnement global
3. Captures d’écran :

Vous devez fournir des captures d’écran de :

la page d’inscription
la page de connexion
l’interface où l’on voit les mariages
l’ajout d’un invité lié à un mariage
l’ajout d’un prestataire lié à un mariage
les tests Postman ou Insomnia pour les routes principales
si possible, un aperçu de MongoDB Compass montrant les données enregistrées
Consignes :
Travaillez proprement et de manière structurée.
Respectez l’architecture du projet vue en classe.
Commentez votre code si nécessaire.
Vérifiez que votre application fonctionne avant de rendre le travail.
Soignez particulièrement la clarté de votre README.
Critères d’évaluation :

Vous serez évalués sur :

le bon fonctionnement de l’inscription et de la connexion
la liaison correcte entre backend et frontend
le bon fonctionnement de l’ajout des mariages, invités et prestataires
la bonne liaison entre les entités
la qualité du code
la qualité de la documentation
la clarté des captures d’écran et des tests API
